Hello, world

The whole install story:

<script src="https://unpkg.com/@velinjs/all/velin-common.min.js"></script>

<div id="app">
  <h1 vln-text="'Hello, ' + name + '!'"></h1>
  <input vln-input="name" />
</div>

<script>
  Velin.bind(document.getElementById('app'), { name: 'World' });
</script>

Why Velin?

State stays a plain object

No hooks, no signals, no .value. Read it, log it, step through it.

No magic syntax

Directives take real JavaScript expressions. What you read is what runs.

Load only what you use

Core is 6.2 KB gzipped. Add router, fragments, or events only if you need them.

The template is the view

HTML holds structure; JavaScript holds state. That separation is not negotiable.
