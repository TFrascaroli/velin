Velin Devtools

An in-page companion for inspecting Velin state trees, directive bindings, and reactive updates as they fire. Currently in alpha — it answers the questions it claims to, but expect rough visuals.

Trying it now

Run the site in dev mode and devtools is injected into every /playground/ page automatically: 

npm run serve:dev
# open http://localhost:3123/playground/hello-world.html
# press Ctrl+Shift+V to toggle the devtools panel

Open any example, then open the same example directly at its /playground/ URL — devtools attaches to whatever Velin instance is running on that page.

What you get

  • A live tree of every reactive state object bound in the page.
  • Per-directive breakdown of which expressions read which state paths.
  • An update log — every property write, in order, with the directives it triggered.

Full reference

The devtools doc covers every tab, the question it answers, snapshot cadence, and how to attach the panel to your own page.

Status

Devtools is 0.1.0-alpha.0. Visual polish and clearer user journeys are the current focus.
